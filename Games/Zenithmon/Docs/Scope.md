# Zenithmon -- Scope

**Document purpose:** The BINDING scope gate for Zenithmon. Anything in Section 1
ships; anything in Section 2 is explicitly cut; anything in neither section is out
of scope until a user decision adds it (see Section 4, Change control). When a
design discussion drifts toward a cut or unlisted feature, redirect: "out of scope."

**Status:** Locked by user decisions on 2026-07-03; ratified with plan approval on
2026-07-08 (source: the approved plan, `zenithmon-pok-mon-nested-puddle.md`).
Recorded as DecisionLog entry ZM-D-001 in [DecisionLog.md](DecisionLog.md).

**What Zenithmon is:** a feature-complete Pokemon Sword/Shield-class
monster-collecting RPG built entirely on Zenith systems -- mainline MECHANICS only,
with ~150 original species and original names everywhere (zero Nintendo IP). All
assets are procedurally generated and baked to disk by tools builds.

---

## 1. What's In (locked scope, verbatim from the plan)

- **Dex:** ~150 original species (original names -- no Nintendo IP), 18 types,
  3-stage evolution lines, rarity tiers.
- **World:** classic 8-gym structure -- home village, ~10 towns/cities, ~12-15
  routes, Victory Road, League. No open Wild Area.
- **Battle core (always in):** type chart, moves/PP/accuracy/crits,
  physical/special split, stat stages, status conditions, catching, switching,
  items, exp/levels, evolution, trainer AI.
- **Battle/progression extras:** abilities, natures, IVs/EVs, weather + terrain
  battle effects, breeding/eggs/daycare. No Dynamax-style gimmick.
- **Post-game:** Champion rematch + Battle Tower (endless streak, rental teams,
  escalating AI).
- No audio. No networking/multiplayer/trading.

Full mechanic-by-mechanic detail (formulas, status list, move-effect coverage,
AI tiers) lives in [GameDesignDocument.md](GameDesignDocument.md); the species
dex families and per-town/route content are TBD at S1 (data core) and S9/S10
(world buildout) respectively.

---

## 2. What's Out (explicit cuts)

Every item below is CUT, not deferred. None of these ship without a new user
decision recorded in [DecisionLog.md](DecisionLog.md):

- **Audio** -- the engine has no audio system; the game is silent by design.
- **Networking / multiplayer / trading** -- single-player only.
- **Dynamax-analog** -- no battle gimmick of any kind (no Mega/Z/Tera analog either).
- **Doubles** -- battle format is singles only. (Struct layout does not preclude
  doubles later, but doubles is out of scope regardless.)
- **Substitute / Encore / Transform / weight-based moves** -- documented move/effect
  cuts; the ~60-kind move-effect enum does not include them.
- **Open Wild Area** -- the world is classic towns + routes only.

---

## 3. Boundary rules (how the scope is built)

These are HARD RULES from the plan, restated here because they bound what any
stage may add:

- **`ZM_` prefix** on all game code (mirrors `DP_`/`CB_`).
- **Original names everywhere** -- species, moves, abilities, towns. Mainline
  mechanics are the reference; no Nintendo names, text, or designs.
- **Game data = compiled `const` C-array tables** in `Source/Data/*.cpp` -- not
  disk assets. The "assets baked to disk" mandate covers meshes/textures/anims only.
- **Baked assets are git-ignored** -- regenerated by tools builds under manifest
  guards; a fresh checkout has no `Assets/` (see [AssetManifest.md](AssetManifest.md)).
- **Builds:** `zenith build Zenithmon` or per-game sln msbuild with `/t:Zenithmon`
  only -- never whole-sln. Tests run via `zenith test Zenithmon` only (the old
  per-game runner scripts are deleted; see DecisionLog entry ZM-D-013).

---

## 4. Change control

Scope changes -- adding anything to Section 1, resurrecting anything from
Section 2, or bending a Section 3 rule -- require a **user decision** recorded as
a new entry in [DecisionLog.md](DecisionLog.md) BEFORE any implementation work.
No agent may widen scope on its own judgment. Shrinking scope (cutting an In item)
is also a user decision and gets the same treatment.

A stage gate is not passed until this file, [Roadmap.md](Roadmap.md), and
[Shortfalls.md](Shortfalls.md) agree with what actually shipped.
